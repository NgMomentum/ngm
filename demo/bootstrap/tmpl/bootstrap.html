AngularJS Bootstrap-UI via NgMomentum

These demo pages highlight the use of NgMomentum (NgM) and MobileSiteOS (MSOS) as the mechanism to load individual, near standard AngularJS Bootstrap-UI v2.5.0 component modules. Standard AngularJS Bootstrap-UI is typically loaded in one shot, as a single file, including templates. Not very efficient if you are only using a few components for any given website. And the framework file is not small. NgM however, loads major components "on demand". Only load the one's you need, when you need them.

NgM Bootstrap-UI Base Code

The base NgM Bootstrap-UI file includes the following modules. These are relatively small, commonly used files which are frequent dependencies for other Bootstrap-UI components. Only 36Kb uncompressed.

  • ng.bootstrap.ui.collapse
  • ng.bootstrap.ui.position
  • ng.bootstrap.ui.debounce
  • ng.bootstrap.ui.tabindex
  • ng.bootstrap.ui.isClass
  • ng.bootstrap.ui.stackedMap
  • ng.bootstrap.ui.multiMap

NgM Dependency Managment

Using NgM and MSOS allows loading JavaScript, HTML and CSS as modules (including std AngularJS modules). Most important, MSOS modules call their own dependencies. Once defined, you don't have to keep track of dependency chains. Complex dependency managment is handled at the module level, unlike other module loading scripts. Thus NgM Bootstrap-UI component modules and dependencies are loaded only once, as needed. Only missing dependencies are loaded in future Bootstrap-UI component calls.
